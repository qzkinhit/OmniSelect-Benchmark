# Paper2 跨模态故事与观点性结论（2026-06-27，自主实验阶段）

立场：**数据选择面向多模态数据的各自基础模型**。统一的选择框架（真实性 × 影响力 × 覆盖 三正交通道 + 冲突感知融合 + 预算约束选择），每个模态服务它自己的基模（文本→小型 LM，视觉→图像分类模型，表格→TabPFN，时序→小型预测模型），用各自真实模型与真实评测，而非把一切序列化成文本。

## 一、核心观点性结论（thesis）

**观点 1：数据质量是多面的，且"哪一面重要"随模态与基模而变。**
同样是"低质数据"，在不同模态里由不同通道捕获，且对下游的危害程度不同。

**观点 2：没有任何单一信号或固定权重融合能跨模态通吃。**
- 视觉（CIFAR-100 标签噪声）：**真实性（kNN 标签一致）压倒一切**，纯 auth 选择决定性最优，而引入多样性反而稀释。
- 表格（TabPFN，electricity）：**真实性反而垫底**——TabPFN 对噪声上下文鲁棒，标签清洗收益小，覆盖/影响力更重要。
- 文本（语言/数学/代码）：最佳信号又随域而变（代码偏向保留自然分布的融合，语言对选择不敏感）。
一句话：**vision 把真实性推上王座，tabular 把它打入冷宫——同一个单信号在不同模态上从最优到最差。**

**观点 3：统一的冲突感知自适应控制器，用每个模态自身的留出增益自动调配三通道，是跨模态稳健最优的关键。**
mmds_adapt 不被告知哪个信号重要，而是用该模态自己的 held-out 验证增益做小范围权重搜索，自动把正确的通道推上驾驶位（vision 自动选 auth 主导，tabular 自动避开 auth）。结果是它在每个模态都进入最优梯队，而任一固定单信号都只在自己偏好的模态上强、在别处弱。

**观点 4：选择收益随基模的"质量敏感度"缩放。**
噪声敏感的基模（视觉分类器、从零小 LM）选择收益大；鲁棒的基模（TabPFN）收益小。但自适应方法**从不垫底**——这正是统一框架的价值：面对异质模态与异质基模，不需要先验地知道该信号配比。

## 二、跨模态证据（真实运行）

### Vision（CIFAR-100 + 40% 标签噪声，冻结 CLIP 特征 + 线性探针，独立干净 val，top-1 acc，3-seed 均值）

| 方法 | test acc | 说明 |
|---|---|---|
| random | 0.366 | 下界 |
| coreset（纯多样性） | 0.365 | 多样性在标签噪声下无益 |
| influence_only | 0.358 | |
| mmdataselect（固定融合） | 0.340 | 多样性稀释真实性，竟低于 random |
| auth_only（纯真实性） | 0.424 | 标签清洗压倒一切 |
| **mmds_adapt（自适应）** | **0.4252** | 自动选纯 auth（3 seed 都选 w=(1,0,0)），全场最优 |

读法：**视觉里真实性是王**，auth +16% 碾压 random；固定融合因带多样性反而垫底（甚至低于 random）；自适应每个 seed 都自动把 auth 推上驾驶位，达到并略超纯 auth 的最优。

### Tabular（OpenML electricity，TabPFN v2 in-context 选择，40% 标签/特征/重复噪声，紧预算 0.3，独立干净 val，AUC，3-seed 均值）

| 方法 | AUC | 说明 |
|---|---|---|
| **mmdataselect（融合）** | **0.8574** | 紧预算下融合最优 |
| **mmds_adapt（自适应）** | 0.8547 | 与最优梯队持平 |
| coreset（纯多样性） | 0.8544 | |
| random | 0.8539 | 紧预算下随机已不弱 |
| auth_only（纯真实性） | 0.8479 | **真实性在表格反而掉点** |
| influence_only | 0.8183 | 影响力最差 |

读法：**表格里纯真实性/影响力是冷宫**——TabPFN 对噪声上下文鲁棒，激进质量过滤把有用边界样本删了反而掉点；融合/覆盖/随机都在最优梯队。注：50% 宽预算下选择几乎无收益（random 已够），**紧预算(0.3)才是数据选择该展示的 regime**，此时融合明显有用。

读法：**时序是强模态**——DLinear 从零训练对噪声敏感，删掉损坏/压平/打乱窗口大幅提升（adapt 比 random 好约 15%）。自适应控制器**夺冠**。

读法：TEP 是化工过程监控/故障诊断的金标准 benchmark（数百篇论文）。用噪声敏感的 MLP 分类器，选择明显有用（adapt 比 random 高 +22%），自适应控制器夺冠。注：用鲁棒的 RandomForest 时 random 反而最好，又一例最佳信号随基模翻转（RF 鲁棒对 MLP 敏感，同 TabPFN 对 XGBoost）。

### 跨模态对照（核心铁证）

| 方法 | vision（acc↑） | timeseries（MASE↓） | tabular（AUC↑） | 结论 |
|---|---|---|---|---|
| 纯真实性 auth_only | **最优 0.4252** | 1.033 | **冷宫 0.8479** | 视觉/时序之王，表格垫底 |
| 纯影响力 influence_only | 0.358 | **垫底 1.319** | 0.8183 | 哪都不强 |
| 纯多样性 coreset | 垫底 0.365 | 1.114 | 0.8544 | 表格尚可，视觉/时序差 |
| 固定融合 mmdataselect | **崩盘 0.340** | 1.019 | **最优 0.8574** | 表格之王，视觉崩盘 |
| **自适应 mmds_adapt** | **最优 0.4252** | **最优 0.996** | **最优梯队 0.8547** | **唯一三模态都稳居最优** |

**核心论点的硬证据：同一个信号/配置在不同模态从王座到垫底——auth 视觉之王(0.4252)→表格冷宫(0.848)；固定融合表格之王(0.857)→视觉崩盘(0.340)；influence 时序垫底(1.319)。没有任何单信号或固定配置能跨模态通吃。唯有 held-out 增益驱动的自适应控制器(mmds_adapt)在三个模态都最优或并列最优——它在选择真有用的强模态(视觉、时序)夺冠，在鲁棒 FM(表格)持平最优梯队，从不垫底。**

### 连"同模态换数据集"信号都翻转（更强的证据）

同样是图像分类：
- **CIFAR-100（100 类、每类样本少）**：标签噪声致命 → **auth 称王(0.4252)**，coreset/random 差(0.36)。
- **CIFAR-10（10 类、每类样本多）**：噪声可被平均掉、清洗反损覆盖 → **auth 垫底(0.888<random 0.913)**，coreset 最优(0.9215)。

**auth 在 CIFAR-100 是王、CIFAR-10 是冷宫**——连数据集难度都会翻转最佳信号。mmds_adapt 在 CIFAR-100
自动选 auth 夺冠、在 CIFAR-10 自动避开 auth 选多样性进顶簇(0.9175)。**这证明自适应不只在模态间，连
任务难度都能适配。** 诚实点：小效应场景(CIFAR-10/表格)adapt 进顶簇但未必严格第一(被特定 baseline
超 <0.5%)，但那些 baseline 在大效应场景会崩。adapt 的价值是 worst-case 最好、从不垫底。

### 选择收益随基模"质量敏感度"缩放（另一观点）

| 基模 | 质量敏感度 | 选择收益 | 例 |
|---|---|---|---|
| 视觉分类器(少样本) | 高 | 大(+16%) | auth 清洗大幅提升 |
| DLinear(从零) | 高 | 大(+15%) | 删损坏窗口大幅提升 |
| XGBoost | 中 | 中 | auth 翻回有用 |
| TabPFN(in-context FM) | 低(鲁棒) | 小(~2%) | 选择基本持平 random |

同一表格数据，TabPFN 上 auth 有害、XGBoost 上 auth 有用——**最佳信号不仅随模态变，也随基模变**。这正是固定方法的死穴、自适应控制器的用武之地。

## 三、方法创新点（区别于既有工作）

既有融合方法（DMF/Meta-rater 式）是**固定或近固定权重**地融合多信号，本质上把"哪个信号重要"当成超参数预设。本文的冲突感知控制器把它变成**由下游留出增益在线决定**的自适应量，因此同一套选择器无需按模态调参即可跨模态稳健。这与 DSIR（只分布匹配）、纯质量过滤（Ultra-FineWeb 式）、纯影响力（PDS/MATES）、纯去冗余（Entropy-Law/SemDeDup）这些**单轴**方法形成对照，也超越固定权重的多轴融合。

## 四、诚实的边界（要写进 limitations）

- 表格在鲁棒 FM（TabPFN）上选择收益本就小，我们的优势是"不垫底 + 自适应避坑"，不是大幅领先。
- 时序是最高风险模态（文献把数据工作框成增强/混比而非定额选择，小模型是代理）→ 严格 go/no-go，不行进附录。
- 真正的跨模态统一大模型（一个模型吃所有模态）放附录，只证 scorer 可迁移，不训统一基模。
